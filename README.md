# Snake Game (Pygame) 

Учебный проект по курсу Python, реализующий классическую игру «Змейка» с использованием библиотеки Pygame.  
Проект демонстрирует работу с графикой, событиями, объектно‑ориентированным программированием, документированием кода, модульным тестированием и генерацией HTML‑документации через Sphinx.

## Возможности

- Главное меню с:
  - вводом имени игрока;
  - выбором начальной скорости;
  - выбором уровня сложности (`Easy`, `Normal`, `Hard`);
  - кнопкой «Старт».
- Игровое поле 15×15 клеток с шахматным фоном.
- Управление змейкой по сетке, рост при поедании яблок.
- Пауза, изменение скорости «на лету», постепенное усложнение (ускорение при наборе очков).
- Звуки поедания яблока и окончания игры.
- Сохранение статистики игр в файл.
- Модульные тесты для классов `Snake` и `Apple`.
- Докстринги в стиле Google + аннотации типов, генерация HTML‑документации через Sphinx.

## Управление

### В главном меню

- Мышь — выбор поля имени, изменение скорости, переключение сложности, нажатие кнопки «Старт».
- Клавиатура — ввод имени в активное поле.
- `Esc` — выход из приложения.

### В игре

- `W` / `↑` — движение вверх.  
- `S` / `↓` — движение вниз.  
- `A` / `←` — движение влево.  
- `D` / `→` — движение вправо.  
- `P` — пауза / продолжить.  
- `+` (включая NumPad) — увеличить скорость.  
- `-` — уменьшить скорость.  
- `Esc` — выйти из игры в меню.

## Логика игры

- Поле состоит из 15×15 клеток. Змейка двигается дискретно по клеткам, а не по пикселям. 
- Яблоко всегда появляется в свободной клетке, не занятой телом змейки.  
- При поедании яблока:
  - счёт увеличивается;
  - длина змейки растёт;
  - периодически увеличивается скорость (интервал зависит от сложности).
- Игра заканчивается, если змейка выходит за границы поля или врезается в своё тело.  

После окончания игры отображается экран Game Over с двумя кнопками:

- «Играть снова» — перезапуск игры с теми же настройками;
- «Выход» — возврат в главное меню.

Результат каждой партии дописывается в `results.txt` строкой вида:
```YYYY-MM-DD HH:MM:SS;ИмяИгрока;Счёт```

## Тестирование

Проект использует стандартный модуль `unittest`.

### Запуск всех тестов
```python -m unittest discover -s tests```

Содержимое тестов:

- `tests/test_snake.py`
  - проверка начальной длины змейки;
  - изменение координат при движении;
  - запрет разворота на 180°;
  - рост при вызове `grow()`;
  - определение самоукуса.
- `tests/test_apple.py`
  - проверка, что `Apple.spawn_random()` не размещает яблоко на клетке змейки.

## Документация (Sphinx)

HTML‑документация генерируется из докстрингов и `.rst`‑файлов с помощью Sphinx.

### Сборка документации
```cd docs```
```make html```
Сгенерированные страницы будут в:
`docs/_build/html/index.html`


Открой этот файл в браузере, чтобы просмотреть документацию по модулям `main`, `game.snake`, `game.apple`, `game.base` и т.д.

## Используемые технологии

- Python 3.x  
- Pygame — графика, окно, обработка событий, отрисовка, шрифты и звуки. 
- unittest — модульное тестирование.  
- Sphinx — генерация HTML‑документации по docstring‑ам.

Проект демонстрирует полный цикл разработки небольшой игры: от архитектуры и логики до тестов и автодокументации.
